C# 初心者向け: ジャグ配列を学ぼう

2次元配列を学んだあとに、実務コードでよく見かける「配列の中に配列が入っている形」を理解するための教材です。

1. ジャグ配列とは

ジャグ配列は、配列の中に、さらに配列を入れる書き方です。 C# では int[][] のように書きます。 

int[][] scores = new int[3][];

scores[0] = new int[] { 80, 90 };
scores[1] = new int[] { 70, 85, 95 };
scores[2] = new int[] { 100 };
読み方のコツ: int[][] は「int の配列の配列」です。 最初の [] が外側の配列、次の [] が内側の配列だと考えるとわかりやすいです。

2. 2次元配列との違い

C# の2次元配列 int[,] は、行と列がきれいな表の形になります。 一方、ジャグ配列 int[][] は、行ごとに長さを変えられます。

種類 型 イメージ アクセス方法
2次元配列 int[,] 全部の行が同じ列数の表 matrix[行, 列]
ジャグ配列 int[][] 配列の中に配列。行ごとに長さが違ってよい scores[行][列]

2次元配列

int[,] table =
{
    { 1, 2, 3 },
    { 4, 5, 6 }
};

2行3列のきれいな表です。

ジャグ配列

int[][] jagged =
{
    new int[] { 1, 2 },
    new int[] { 3, 4, 5 },
    new int[] { 6 }
};

1行目は2個、2行目は3個、3行目は1個です。

3. JS / TS の配列と比べる

JavaScript や TypeScript の経験があるなら、ジャグ配列は number[][] にかなり近いです。

TypeScript

const scores: number[][] = [
  [80, 90],
  [70, 85, 95],
  [100]
];

console.log(scores[1][2]); // 95

C#

int[][] scores =
{
    new int[] { 80, 90 },
    new int[] { 70, 85, 95 },
    new int[] { 100 }
};

Console.WriteLine(scores[1][2]); // 95
大事な違い: JS の配列はあとから型が混ざりやすいですが、C# の int[][] は 「中身は int の配列」と決まっています。文字列を入れることはできません。

4. 作り方

最初から値を入れて作る

string[][] classes =
{
    new string[] { "佐藤", "鈴木" },
    new string[] { "田中", "高橋", "伊藤" },
    new string[] { "山本" }
};

外側だけ先に作る

int[][] scores = new int[3][];

scores[0] = new int[] { 80, 90 };
scores[1] = new int[] { 70, 85, 95 };
scores[2] = new int[] { 100 };

new int[3][] は「外側の配列は3個分ある」という意味です。 ただし、内側の配列はまだ作られていません。

注意: 内側の配列を作る前に scores[0][0] のように使うと、 NullReferenceException になります。 
int[][] scores = new int[3][];

// scores[0] がまだ null なのでエラーになる
Console.WriteLine(scores[0][0]);

5. 値の取り出し方

ジャグ配列は scores[行][列] で値を取り出します。 2次元配列の matrix[行, 列] とはカッコの形が違います。 

int[][] scores =
{
    new int[] { 80, 90 },
    new int[] { 70, 85, 95 },
    new int[] { 100 }
};

Console.WriteLine(scores[0][1]); // 90
Console.WriteLine(scores[1][2]); // 95
Console.WriteLine(scores[2][0]); // 100

6. for 文で全部表示する

ジャグ配列では、外側の配列の長さと、内側の配列の長さをそれぞれ見ます。 2次元配列の GetLength(0) や GetLength(1) ではなく、 普通の配列と同じ Length を使います。 

int[][] scores =
{
    new int[] { 80, 90 },
    new int[] { 70, 85, 95 },
    new int[] { 100 }
};

for (int row = 0; row < scores.Length; row++)
{
    for (int col = 0; col < scores[row].Length; col++)
    {
        Console.Write(scores[row][col] + " ");
    }

    Console.WriteLine();
}

出力はこうなります。

80 90
70 85 95
100
ポイント: scores.Length は外側の配列の長さです。 scores[row].Length は、今見ている行の内側の配列の長さです。

7. foreach で全部表示する

foreach で書くと、「外側の配列から1行ずつ取り出す」 「その行の中身を1個ずつ取り出す」という流れになります。 

foreach (int[] row in scores)
{
    foreach (int score in row)
    {
        Console.Write(score + " ");
    }

    Console.WriteLine();
}

int[] row の部分が大事です。 外側から取り出されるのは int ではなく、int[] です。

8. よくある使いどころ

  • クラスごとに生徒数が違う
  • 日ごとに記録したデータ数が違う
  • ゲームで、ステージごとに敵の数が違う
  • API から来たデータが「配列の中に配列」の形になっている

全部の行の列数が同じなら int[,] でもよいです。 行ごとに長さが違う可能性があるなら、ジャグ配列のほうが自然です。

9. 練習問題

問題1: 値を取り出す

次のコードで Console.WriteLine(numbers[1][0]); は何を表示しますか。

int[][] numbers =
{
    new int[] { 10, 20 },
    new int[] { 30, 40, 50 },
    new int[] { 60 }
};
答え: 30
numbers[1] は2行目の { 30, 40, 50 }、 その [0] なので 30 です。

問題2: 合計を出す

全部の点数の合計を出すコードを書いてみましょう。

int[][] scores =
{
    new int[] { 80, 90 },
    new int[] { 70, 85, 95 },
    new int[] { 100 }
};
解答例: 
int total = 0;

for (int row = 0; row < scores.Length; row++)
{
    for (int col = 0; col < scores[row].Length; col++)
    {
        total += scores[row][col];
    }
}

Console.WriteLine(total); // 520

問題3: TypeScript と見比べる

次の TypeScript と同じ意味になる C# のコードを書いてみましょう。

const names: string[][] = [
  ["Aki", "Ben"],
  ["Coco"],
  ["Dai", "Ema", "Futa"]
];
解答例: 
string[][] names =
{
    new string[] { "Aki", "Ben" },
    new string[] { "Coco" },
    new string[] { "Dai", "Ema", "Futa" }
};

10. まとめ

  • ジャグ配列は int[][] のように書く。
  • 意味は「配列の中に配列」。
  • 値は scores[行][列] で取り出す。
  • 行ごとに長さを変えられる。
  • JS / TS の number[][] とかなり近い。
  • 2次元配列 int[,] は matrix[行, 列] で、きれいな表向き。
